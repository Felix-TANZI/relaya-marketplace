5. NOTIFICATIONS CLIENT
-----------------------

13. CONFIGURATION VITE / FRONTEND
---------------------------------
- Ajout d'une option `VITE_CACHE_DIR` dans la configuration Vite.
- Cela permet d'utiliser un cache alternatif si `node_modules/.vite` n'est pas accessible en écriture.
- Utile pour contourner certains problèmes de permission en local.


14. BASE DE DONNEES / DEPLOIEMENT BACKEND
-----------------------------------------
- Analyse de l'erreur Django :
  "failed to resolve host pg-38426783-junsts719-37a3.c.aivencloud.com".
- Conclusion : le backend utilisait un ancien host Aiven.
- La base de données était allumée, mais le host configuré côté Render était incorrect.
- Ajout du support `DATABASE_URL` dans la configuration Django.
- La configuration Django lit maintenant `DATABASE_URL` en priorité.
- Ajout du support SSL via `sslmode=require`.
- Ajout d'un timeout de connexion PostgreSQL.
- Mise à jour de `.env.example` avec :
  - DATABASE_URL,
  - POSTGRES_SSLMODE,
  - POSTGRES_CONNECT_TIMEOUT.
